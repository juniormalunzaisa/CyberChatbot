 # Project README

## Project Overview

This project is a multi-part software development assignment designed to demonstrate programming, problem-solving, and version control skills. The application was developed incrementally across several parts, with each section building upon the previous one. The project demonstrates the use of programming logic, structured development, and proper source control practices using GitHub.

The goal of the project is to design and implement a functional application while applying concepts such as user input handling, program structure, and clear documentation. Continuous improvements and testing were applied throughout development to ensure that the final application functions correctly and meets the required specifications.

---

## Project Structure

The project is divided into three main parts:

### Part 1

Part 1 focuses on building the foundation of the application. In this section, the basic structure of the program was created, including the core functionality and user interaction. The application accepts user input, processes it, and produces appropriate responses. This stage establishes the initial logic and framework used throughout the rest of the project.

### Part 2

Part 2 expands on the initial system by adding additional features and improvements. Enhancements were made to the logic, structure, and user interaction to make the program more functional and user-friendly. The codebase was also organized into classes and components to improve readability and maintainability.

### Final Part (Part 3)

The final part integrates all previous work into a complete and functional application. Any remaining functionality was implemented and the program was tested to ensure stability and correctness. This stage focuses on refining the system, ensuring that all features operate together properly, and preparing the project for final submission.

---

## GitHub and Version Control

The project was managed using GitHub to maintain version control and track development progress.

Key requirements followed:

* Each part of the project was pushed to GitHub.
* A minimum of **six commits** were made for each part of the project.
* Commits were used to document progress and track changes made during development.
* Continuous Integration (CI) was implemented using **GitHub Actions Workflow** to automate checks and ensure the project builds correctly.

Using GitHub ensured proper version management and provided a clear development history.

## Submission Requirements

Each part of the project is submitted with the corresponding GitHub link. The submission includes:

* Part 1 source code

## Presentation Requirement

A video presentation accompanies the final submission. The purpose of the presentation is to demonstrate the functionality of the application and explain the code implementation.

The presentation includes:

* A walkthrough of the application
* Demonstration of the program running
* Explanation of the major components and logic
* Description of how the program was developed across the different parts

Important presentation guidelines:

* The video must include a **voice-over using the student's own voice**
* **AI-generated voices are not allowed**
* The video should clearly explain how the code works
* The video must be uploaded as an **unlisted YouTube video** (or similar platform)
